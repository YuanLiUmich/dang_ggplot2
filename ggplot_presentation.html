Layers

ggplot2 uses the idea of layers and the operator \(+\) to add layers:

Coordinates and variables

ggplot(annual_counts, aes(x=year, y=count))

Plotting type

ggplot(annual_counts, aes(x=year, y=count)) +
  geom_line()

Each type of plotting is a new layer:

ggplot(annual_counts, aes(x=year, y=count)) +
    geom_line() +
    geom_point()


Aesthetics

From my experience, the most frequently used and helpful arguments for adjusting the labels and axes are:

  • scale_x_continuous(): can specify range and break
  • scale_y_continuous()
  • labs(): easiest way to set x, y, and main titles
  • scale_color_manual(): can specify colors and how legend is shown with great freedom
ggplot(annual_counts, aes(x=year, y=count)) +
    geom_point() +
    scale_y_continuous(limits=c(0,NA)) +
    scale_x_continuous(breaks=c(1990, 1995, 2000, 2005, 2010, 2015)) +
    labs(x="Year",
         y="Number of cases",
         title="The number of Lyme disease cases has been rising since 1990") +
    theme_classic()

ggplot(metadata_pcoa, aes(x=axis1, y=axis2, color=dx)) +

        values=c("#00AFBB", "#E7B800", "#FC4E07"),

    theme_classic()

One thing I dont like about ggplot2 is that the significance needs to be calculated beforehand and added as a drawing when the statistics is slightly complicated:

meta_alpha_box <- 
  ggplot(meta_alpha, aes(x=diagnosis, y=fit_result, color=diagnosis)) +
    geom_boxplot() +

Package ggpubr can help:

  • compare_means(): to performs one and multiple mean comparisons.
  • stat_compare_means(): to automatically add p-values and significance levels to a ggplot.

To add p-values between groups, you need to specify the comparisons you want:

library(ggpubr)

compare_means(fit_result~diagnosis, data=meta_alpha, method = "t.test", paired = FALSE)

Ordering

Package forcats is very helpful when you want to put categorical variables in a specific order:

  • fct_reorder(): reordering a factor by another variable.
  • fct_infreq(): reordering a factor by the frequency of values.
  • fct_relevel(): changing the order of a factor by hand.
library(forcats)

ggplot(starwars, aes(x = eye_color)) + 
  geom_bar() + 
  coord_flip()

starwars %>%
  mutate(eye_color = fct_infreq(eye_color)) %>%
  ggplot(aes(x = eye_color)) + 
  geom_bar() + 
  coord_flip()


Arrangement

Sometimes you would want to arrange multiple plots in one figure, and the plots dont always fit into a grid. I found packages patchwork and ggarrange useful. I would try patchwork first, as it is better maintained, and use ggarrange if patchwork doesnt work.

You can freely define the layout, including blank spaces.

library(patchwork)

p1 <- ggplot(mtcars) + geom_point(aes(mpg, disp))
p2 <- ggplot(mtcars) + geom_boxplot(aes(gear, disp, group = gear))
p3 <- ggplot(mtcars) + geom_smooth(aes(disp, qsec))
p4 <- ggplot(mtcars) + geom_bar(aes(carb))

layout <- '
A#B
#C#
D#E
'
wrap_plots(A = p1, B = p2, C = p3, D = p4, design = layout)
